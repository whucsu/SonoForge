# Этап 1. Стратегическая концепция MVP: Персональный эхокардиографический аналитический инструмент

## 1. Стратегическое позиционирование и целевое назначение

### 1.1. Назначение инструмента
Приложение разрабатывается как **персональный десктопный инструмент для собственных нужд** врача-эхокардиографиста. Это не медицинское изделие (SaMD), не исследовательский инструмент (RUO), не коммерческий продукт. Это **личная рабочая станция** для анализа эхокардиографических исследований, экспортированных с УЗ-аппарата.

### 1.2. Ключевые отличия от предыдущей концепции
| Аспект | Было (RUO) | Стало (Персональный инструмент) |
|---|---|---|
| **Регуляторный статус** | Research Use Only | Для собственных нужд (не является ни RUO, ни SaMD) |
| **Терминология** | «Коэффициент сократимости», «Параметры релаксации» | Клинические термины: LVEF, КДО/КСО, E, A, e', s', E/e', VTI |
| **Водяные знаки** | Обязательный неразрывный watermark | Отсутствуют |
| **Модальные дисклеймеры** | При запуске и экспорте | Отсутствуют |
| **Нормативная база** | Только ASE 2015 | ASE 2015 + ASE 2016 + EACVI Recommendations |
| **Сегментация** | Snakes (scikit-image) | ONNX-модели (CPU, Фаза 2) + ручной контур / сплайн-редактор |
| **Спекл-трекинг** | Оптический поток (OpenCV CUDA) | Исключён из проекта |
| **Интеграция с PACS** | DICOMweb (REST API) | Локальные папки / сетевые диски |

### 1.3. Рабочий процесс с данными
Данные поступают через **локальные папки или сетевые диски**, куда УЗ-аппарат экспортирует исследования. Приложение **не выполняет деидентификацию** — работает с файлами как есть. В браузере исследований группировка по **дате и времени исследования** (`StudyDate` + `StudyTime`), без использования персональных идентификаторов.

## 2. Функциональный скоп: MVP, Фаза 1 и Фаза 2

### 2.0. Полный MVP (целевое состояние)

- **Форматы:** DICOM (2D, Color, PW, CW, TDI), MP4, JPEG/PNG
- **Просмотр:** 2D viewer + side-by-side sync (`MasterClock`)
- **Допплер (PW / CW / TDI):** AT, DT, IVRT, VTI, Vpeak, Vmean, PGpeak, PGmean, E/A, e' septal, e' lateral, E/e'
- **2D измерения:** толщина стенок (IVSd, LVPWd), ЛВОТ (LVOT)
- **Расчёты ЛЖ:** Тей-Хольц, Симпсон (биплан + моноплан), Area-Length
- **Расчёты камер:** ЛЖ, ЛП, ПП — объёмы и функция
- **Функция ПЖ:** TAPSE, s' ПЖ, FAC ПЖ
- **Сегментация:** ONNX (CPU) + сплайн-редактор
- **Диастолическая функция:** полный блок ASE 2016 (включая LAVmax, TR Vmax, LVM, градации)
- **Фазы цикла:** ручные hotkeys ED/ES + парсинг ЭКГ из DICOM *(Фаза 2)*
- **Отчётность:** PDF + CSV/JSON

### 2.1. Фаза 1 (первый рабочий релиз)

| Категория | Состав |
|---|---|
| **Данные** | DICOM + MP4 + JPEG/PNG |
| **Просмотр** | 2D viewer (play/pause, W/L, zoom, pan) |
| **Допплер** | PW / CW / TDI: AT, DT, IVRT, VTI, Vpeak, Vmean, PGpeak, PGmean, E/A, e' septal, e' lateral, E/e' |
| **2D измерения** | Толщина стенок, ЛВОТ, линейные калиперы |
| **Расчёты** | Тей-Хольц; Симпсон биплан + моноплан + Area-Length — **ручное создание контура** |
| **Фазы цикла** | Ручная маркировка ED/ES (hotkeys) |
| **Не входит** | Side-by-side, ONNX, сплайн-редактор, ЛП/ПП/ПЖ, полный ASE 2016, PDF, ECG |

### 2.2. Фаза 2 (расширение до полного MVP)

Всё из §2.0, что не вошло в Фазу 1:
- Side-by-side sync (`MasterClock`)
- ONNX-сегментация + сплайн-редактор
- Объёмы ЛП, ПП; TAPSE, s' ПЖ, FAC ПЖ
- Полный блок диастолической функции ASE 2016 (LAVmax, TR Vmax, LVM, градации)
- Парсинг ЭКГ из DICOM для ED/ES
- PDF + CSV/JSON
- AI-валидация (±3% vs EchoPAC)

## 3. Работа с данными и визуализация

### 3.1. Локальный браузер исследований
- Рекурсивный сканер выбранной папки
- Дерево: **«Дата/Время исследования → Серия (модальность) → Экземпляр»**
- При сканировании: `StudyDate`, `StudyTime`, `StudyInstanceUID`, `SeriesDescription`, `Modality`, `NumberOfFrames` — **без** `PatientName` / `PatientID`
- Метаданные без загрузки пикселей

### 3.2. Окно просмотра 2D
- Воспроизведение мультифреймовых петель, контроль FPS
- Window/Level, зум, панорамирование
- **Фаза 1:** ручное обведение контура (полигон / сплайн) для Симпсона и Area-Length
- **Фаза 2:** ONNX Auto Segment + сплайн-редактор; side-by-side с `MasterClock`

### 3.3. Окно спектрального допплера (PW / CW / TDI)

**Типы маркеров:**

| Тип | Параметры | UI |
|---|---|---|
| **Пиковый** (1 клик) | E, A, e' sept, e' lat, a', s', Vpeak | Клик на пик → скорость в см/с |
| **Интервальный** (2 клика) | DT, IVRT, AT | Начало и конец интервала на оси времени |
| **Трассировка** (polyline) | VTI | Ручное обведение огибающей вдоль спектра; интеграл методом трапеций |

**CW Doppler** — тот же UI, что PW/TDI. Измеряемые параметры: **Vpeak, Vmean, VTI, PGpeak, PGmean** (например, МР, АР, ТР, АС). Пиковые маркеры для Vpeak; трассировка для VTI; PG рассчитывается по Бернулли (4v²).

> Автоматическая трассировка огибающей **исключена**. VTI — только ручная трассировка.

### 3.4. Инструменты калибровки
- **Автокалибровка:** `PixelSpacing` / `ImagerPixelSpacing` из DICOM (через `DicomMetadataMapper` → domain dataclass)
- **Ручная калибровка:** линия по шкале глубины
- **Универсальный калипер:** линейные, угловые, площадные измерения

### 3.5. Привязка к фазам сердечного цикла
- **Фаза 1:** hotkeys ED (`D`) / ES (`S`)
- **Фаза 2:** парсинг ЭКГ из DICOM Waveform Sequence (R-зубец) с откатом на ручной режим

## 4. Количественный анализ

### 4.1. Геометрия и объёмы (ASE 2015 / EACVI)
- **Линейные:** LVEDD, LVESD, IVSd, LVPWd, LA dimension, LVOT
- **Объёмы ЛЖ:** Симпсон биплан (A4C + A2C), моноплан, Area-Length, Тей-Хольц
- **Объёмы ЛП, ПП** *(Фаза 2)*: Area-Length / Симпсон
- **LVEF:** `(EDV - ESV) / EDV × 100%`
- **LVM** *(Фаза 2)*: по формуле ASE (толщина стенок + LVEDD)

### 4.2. Функция ПЖ *(Фаза 2)*
- TAPSE (M-mode / 2D калипер)
- s' ПЖ (TDI)
- FAC ПЖ (end-diastolic / end-systolic area)

### 4.3. Диастолическая функция (ASE 2016)
- **Фаза 1:** E, A, E/A, DT, IVRT, AT, e' sept/lat, E/e', VTI, PG
- **Фаза 2:** LAVmax, TR Vmax, LVM, индексированные объёмы, градации нарушения диастолической функции

### 4.4. ИИ-сегментация *(Фаза 2)* — EchoNet Segmentation Lite

**Утверждённая стратегия:** сегментационная ветка [EchoNet-Dynamic](https://github.com/echonet/dynamic) (MIT), **без** 3D видео-моделей EF.

| Уровень | Модель | Когда использовать |
|---|---|---|
| **Базовая** | DeepLabV3-ResNet50 (`deeplabv3_resnet50_random.pt` → ONNX) | RAM ≥ 16 GB, целевой CPU i5/Ryzen 5+ |
| **Lite** | DeepLabV3-MobileNetV3-Large (fine-tune / distillation на EchoNet) | RAM 8 GB или инференс > 500 мс |
| **Запасная** | CAMUS U-Net 1 (MIT) | Benchmark / fallback при domain gap |

**Не используем:** R2+1D / R3D / MC3 видео-модели EchoNet (прямой EF — конфликт с Симпсоном, высокий RAM).

**Пайплайн:**
1. Инференс **только на кадрах ED и ES** (не на всём видео)
2. ONNX на CPU (`OnnxInferenceEngine` в Infrastructure)
3. Domain: `segmentation_service` — resize 112×112, нормализация, маска → контур → масштаб в координаты кадра
4. Сплайн-редактор → пересчёт Симпсона
5. Опционально: ONNX INT8 dynamic quant; `ProcessPoolExecutor` с **1 worker**

Детали — Приложение В.

### 4.5. Отчётность *(Фаза 2)*
- PDF (`ReportLab`), CSV/JSON
- Таблицы измерений, ключевые кадры с оверлеями, референсные значения ASE/EACVI

## 7. Архитектурные принципы

### 7.1. Clean Architecture
**Presentation → Application → Domain ← Infrastructure**

| Слой | Ответственность |
|---|---|
| **Presentation** | UI, QImage, PyQtGraph оверлеи. Без pydicom/cv2/onnxruntime |
| **Application** | AppController, StateManager, MasterClock (Ф2), Workers, ReportService (Ф2) |
| **Domain** | Dataclass-модели, расчёты, `segmentation_service` (NumPy). **Без** pydicom, onnxruntime |
| **Infrastructure** | Сканер, `DicomMetadataMapper`, `DicomReaderImpl`, `OnnxInferenceEngine`, EcgWaveformParser (Ф2) |

**DICOM-метаданные:** Infrastructure (`DicomMetadataMapper`) читает `pydicom.Dataset` и маппит в domain `StudyMetadata` / `SeriesMetadata` / `InstanceMetadata`. Domain не знает о pydicom.

**ONNX:** инференс — Infrastructure (`OnnxInferenceEngine`); пред/постобработка — Domain (`segmentation_service`).

### 7.2. Рендеринг (PyQtGraph)
`GraphicsLayoutWidget` + `ViewBox`: `ImageItem` (кадр) + `PlotDataItem` / `ROI` / `ScatterPlotItem` (контуры, калиперы, маркеры). Doppler — `PlotWidget` с тем же подходом.

### 7.3. Память
- Worker → `QImage` → Signal → Main Thread
- Ring Buffer 50 кадров для MP4

## 12. Приложение В. ONNX-модель сегментации ЛЖ (утверждено)

### 12.1. Решение: EchoNet Segmentation Lite

**Репозиторий:** [echonet/dynamic](https://github.com/echonet/dynamic) · **Лицензия:** MIT · **Стоимость:** бесплатно

EchoNet-Dynamic содержит три компонента; для проекта используется **только первый**:

| Компонент EchoNet | Архитектура | Используем? | Причина |
|---|---|---|---|
| Сегментация ЛЖ | DeepLabV3 + ResNet50 | **Да** | Контур эндокарда → Симпсон → LVEF |
| EF из видео | R2+1D / R3D / MC3 | **Нет** | Прямой EF, без контура; OOM на слабых ПК |
| Кардиомиопатия | Beat-by-beat | **Нет** | Вне скопа MVP |

**Веса (базовая модель):** `deeplabv3_resnet50_random.pt`  
**URL:** [v1.0.0 release](https://github.com/douyang/EchoNetDynamic/releases/download/v1.0.0/deeplabv3_resnet50_random.pt)  
**Экспорт:** PyTorch → ONNX (opset 17), `onnxruntime` CPU provider

### 12.2. Контракт модели

| Параметр | Значение |
|---|---|
| Вход | `(1, 3, 112, 112)` float32, нормализация mean/std как в EchoNet |
| Выход | Бинарная маска ЛЖ `(112, 112)` или logits → sigmoid |
| Постобработка | `mask_to_contour()` → масштаб в исходное разрешение кадра |
| Триггер | Только кадры ED и ES (после hotkeys `D` / `S`) |
| Целевой view | A4C (основной); A2C — с ожидаемой ручной коррекцией |

### 12.3. Стратегия против OOM

| RAM | Действие |
|---|---|
| **≥ 16 GB** | ResNet50 ONNX; при необходимости INT8 quant |
| **8 GB** | MobileNetV3-Lite ONNX **или** ResNet50 INT8 + 1 worker |
| **< 8 GB** | AI опционален; основной путь — ручной контур (Фаза 1) |

**Оптимизации (обязательные к проверке на Tier 1):**
- INT8 dynamic quantization (`onnxruntime.quantization`)
- `ProcessPoolExecutor(max_workers=1)` — один процесс инференса
- Не загружать 3D-ветки EchoNet
- Явное `del` тензоров после `mask_to_contour()`

### 12.4. Lite-вариант (MobileNetV3)

Если ResNet50 не укладывается в **< 500 мс** или даёт OOM:
- Архитектура: `deeplabv3_mobilenet_v3_large` (torchvision)
- Обучение: fine-tune на EchoNet-Dynamic dataset **или** distillation от ResNet50
- Лицензия: MIT (torchvision + собственные веса)
- Целевой размер ONNX: **< 30 MB**

### 12.5. Запасные кандидаты (не основной путь)

| Источник | Лицензия | Когда |
|---|---|---|
| [CAMUS U-Net 1](https://github.com/GKalliatakis/camus-segmentation-pytorch) | MIT | Benchmark; лёгкий fallback |
| [nnunet_camus_cv1_onnx](https://huggingface.co/gillesvdv/nnunet_camus_cv1_onnx) | Академическая | Только если CAMUS-domain достаточен; тяжёлая для 8 GB |

**Исключено:** EchoROI (ROI-кроп, не эндокард), Trillion.ai (лицензия не MIT).

### 12.6. Экспорт в ONNX (скрипт)

```bash
# Один раз: pip install torch torchvision onnx onnxruntime
python scripts/export_echonet_seg_to_onnx.py --verify --quantize-int8
```

Скрипт:
- скачивает `deeplabv3_resnet50_random.pt` в `models/` (если нет локально);
- экспортирует `models/echonet_seg_resnet50.onnx` (+ опционально `*_int8.onnx`);
- обновляет `models/model_manifest.json` (SHA256, размер, `exported_at`).

### 12.7. План верификации (до Спринта 5)

1. Запустить `scripts/export_echonet_seg_to_onnx.py --verify`
2. Прогон 5–10 A4C DICOM (Tier 1): peak RAM, время кадра, визуальная оценка контура
3. LVEF (AI-контур + Симпсон) vs ручной контур и EchoPAC — критерий ±3%
4. При fail по RAM/времени → `*_int8.onnx` или MobileNetV3-Lite
5. При калибровке нормализации — записать `fixed_mean` / `fixed_std` в manifest

```text

├── .gitignore

├── deeplabv3_resnet50_random.pt     # локально, не в git

├── echonet_seg_resnet50_int8.onnx   # опционально
```
