# Этап 3. Проектирование UI и рабочих процессов (Персональная эхокардиографическая станция)

**Статус:** Утверждено для перехода к прототипированию и верстке  
**Фокус:** Клиническая точность (ASE/EACVI), локальный workflow, поэтапная поставка (Фаза 1 → Фаза 2).  
**Исключено:** Веб-интерфейсы, DICOMweb, RUO-ограничения, автоматическая трассировка допплера, спекл-трекинг, CUDA, модули деидентификации.

---

## 1. Стратегические принципы проектирования UI

1. **Клиническая терминология:** LVEF, КДО, КСО, E, A, e', E/e', VTI, DT, IVRT, AT, PGpeak, PGmean, TAPSE, FAC.
2. **Минимум кликов:** hotkeys для ED/ES, маркеров, play/pause; ИИ-сегментация — одна кнопка *(Фаза 2)*.
3. **Отзывчивость (60 FPS):** единый стек **PyQtGraph** — `ImageItem` + `PlotDataItem` / `ROI` / `ScatterPlotItem` в одном `ViewBox`. **QGraphicsView не используется.**
4. **Локальная скорость:** дерево из метаданных без загрузки пикселей.

## 3. Архитектура главного окна (Layout)

### 3.1. Режим Фазы 1 (одиночный viewer)

* **Левая панель (20%):** браузер исследований
* **Центр (60%):** Viewer — переключение 2D / Doppler
* **Правая панель (20%):** инструменты + результаты измерений
* **Низ:** таймлайн, Play/Pause, FPS, индикатор ED/ES

### 3.2. Режим Фазы 2 (side-by-side)

* **Центр (60%):** два `ViewerWidget` рядом, синхронизированных через `MasterClock`
  - Режим **time-to-time:** `current_time_ms` общий для обоих окон (разный FPS)
  - Режим **frame-to-frame:** синхронизация по номеру кадра
* Переключатель режима синхронизации — в нижней панели
* Остальной layout без изменений

## 4. Ключевые модули UI

### 4.1. Локальный браузер исследований

* Рекурсивный сканер локальной директории / сетевого диска
* Дерево: **`Дата и время исследования` → `Модальность / Серия` → `Экземпляр`**
* Метаданные при сканировании: `StudyDate`, `StudyTime`, `StudyInstanceUID`, `SeriesDescription`, `Modality`, `NumberOfFrames`
* **Не используются:** `PatientName`, `PatientID` — приложение не опирается на персональные идентификаторы
* Thumbnail при выборе серии (первый / средний кадр)

### 4.2. Окно просмотра 2D

* Window/Level, зум, pan, инверсия
* **Фаза 1 — ручной контур:**
  1. Маркировка ED (`D`) / ES (`S`)
  2. Инструмент «Контур» — полигон / сплайн по эндокарду
  3. Мгновенный пересчёт EDV, ESV, LVEF (Симпсон / Area-Length / Тей-Хольц)
* **Фаза 2 — EchoNet Segmentation Lite + сплайн:**
  1. После маркировки ED/ES (`D` / `S`) — `Auto Segment` (hotkey `I`) на **текущем кадре**
  2. ONNX DeepLabV3 (ResNet50 или MobileNetV3-Lite) в `ProcessPoolExecutor` → маска → контур
  3. Контур на `PlotDataItem`; узлы — `ScatterPlotItem`, перетаскивание → пересчёт Симпсона
  4. При OOM / timeout — сообщение «используйте ручной контур» (Фаза 1 fallback)
* **Калиперы:** `LineROI` — LVEDD, LVESD, IVSd, LVPWd, LVOT

### 4.3. Окно спектрального допплера (PW / CW / TDI)

Спектрограмма в `PlotWidget`. Три инструмента:

#### A. Пиковый маркер (`M` + выбор типа)
Один клик на пик → скорость в см/с.

| Режим | Маркеры |
|---|---|
| PW (трансмитральный) | E, A |
| TDI | e' sept, e' lat, a', s' |
| CW / PW (гемодинамика) | Vpeak |

Авторасчёт: E/A, e'/a', E/e' (среднее), PGpeak = 4×Vpeak².

#### B. Интервальный маркер (`T` + выбор типа)
Два клика (начало → конец) на оси времени:

| Маркер | Клинический смысл |
|---|---|
| **DT** | От пика E до возврата к baseline (или до конца замедления) |
| **IVRT** | От закрытия аортального клапана до начала E |
| **AT** | Время нарастания потока (напр. ЛВОТ, стеноз АК) |

#### C. Трассировка VTI (`V`)
- Пользователь **вручную** обводит огибающую спектра полилинией (произвольное число точек)
- Двойной клик / Enter — завершить трассировку
- Площадь под кривой → **VTI** (интеграл трапеций)
- Для CW: та же трассировка; дополнительно **Vmean** = VTI / AT при наличии интервала AT

> Автоматическая трассировка огибающей исключена.

### 4.4. Панель измерений (MeasurementPanel)

**Фаза 1:**
* Геометрия ЛЖ: LVEDD, LVESD, IVSd, LVPWd, LVOT
* Объёмы: EDV, ESV, LVEF (Симпсон / Тей-Хольц / Area-Length — с указанием метода)
* Допплер: E, A, E/A, DT, IVRT, AT, e' sept, e' lat, E/e', VTI, Vpeak, Vmean, PGpeak, PGmean

**Фаза 2 (дополнительно):**
* LVM, LAVmax, TR Vmax
* ЛП, ПП: объёмы
* ПЖ: TAPSE, s', FAC
* Диастолическая функция ASE 2016: градации (норма / I–III)

## 5. Рабочие процессы (UX Scenarios)

### Сценарий А: LVEF — ручной контур (Фаза 1)
1. Выбор серии A4C в браузере
2. ED (`D`) → ES (`S`)
3. Инструмент «Контур» — обведение эндокарда в ED и ES
4. Панель: EDV, ESV, LVEF (моноплан Симпсон)
5. Повтор для A2C → автопереключение на **биплан Симпсон**

### Сценарий А2: LVEF — EchoNet Segmentation Lite (Фаза 2)
1. Шаги 1–2 как в сценарии А (ED/ES hotkeys)
2. На кадре ED: `I` → ONNX-контур эндокарда (DeepLabV3)
3. На кадре ES: повторить `I`
4. Коррекция сплайн-узлами (особенно A2C)
5. Биплан Симпсон после обработки A2C

### Сценарий Б: Диастолическая функция (Фаза 1)
1. PW: пик E → пик A → E/A
2. Интервал DT (2 клика), IVRT (2 клика)
3. TDI sept: пик e' → TDI lat: пик e' → среднее E/e'
4. При необходимости: VTI-трассировка на PW

### Сценарий В: CW — градиенты (Фаза 1)
1. Открыть CW-серию (напр. трикуспидальная регургитация)
2. Пиковый маркер Vpeak → PGpeak = 4×V²
3. Трассировка VTI вдоль спектра → Vmean, PGmean
4. Для стеноза АК: интервал AT на ЛВОТ PW/CW

### Сценарий Г: LVM, LAVmax, TR Vmax (Фаза 2)
1. 2D: LVEDD + IVSd + LVPWd → LVM (ASE cube formula)
2. A4C ES: контур ЛП → LAVmax (Area-Length / Simpson)
3. CW TR: Vpeak → TR Vmax, оценка СДЛА

### Сценарий Д: Функция ПЖ (Фаза 2)
1. TAPSE — линейный калипер на M-mode / 2D
2. TDI lateral tricuspid: s'
3. A4C RV: FAC = (EDA − ESA) / EDA × 100%

### Сценарий Е: Side-by-side (Фаза 2)
1. Открыть две серии (напр. rest / stress или A4C / A2C)
2. Включить side-by-side; выбрать sync: time-to-time или frame-to-frame
3. `MasterClock` синхронизирует скраббер и play
4. ED/ES в одном окне — опционально зеркалирование фазы во втором

### Сценарий Ж: ЭКГ-синхронизация (Фаза 2, запланировано)
1. При наличии Waveform Sequence в DICOM — автоматическое определение R-зубцов
2. Предложение ED/ES по ЭКГ; врач подтверждает или корректирует hotkeys
3. При отсутствии ЭКГ — только ручной режим (без ошибки)

```python
markers_changed = Signal(DopplerMeasurementDTO)
```

```python
def compute(dto: DopplerMeasurementDTO) -> DopplerResults:
  # E, A, E/A из peaks

  # PGpeak = 4 * Vpeak^2
  ...
```

### 8.3. StateManager → MeasurementPanel

```python
measurements_updated = Signal(MeasurementSnapshot)
# MeasurementSnapshot агрегирует DopplerResults + LvefResult + LinearMeasurements
```
