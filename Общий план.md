# План разработки персонального эхокардиографического аналитического инструмента (Python + VS Code)

**Статус проекта:** Персональный десктопный инструмент для клинического анализа эхокардиографических исследований  
**Регуляторный статус:** Для собственных нужд (не является медицинским изделием, не предназначен для коммерческого распространения)  
**Нормативная база:** ASE 2015 (Quantification of Chambers), ASE 2016 (Diastolic Function), EACVI Recommendations  
**Архитектура:** Локальное десктопное приложение (без веб-интерфейса и DICOMweb)  
**Поставка:** Фаза 1 (ручные измерения) → Фаза 2 (AI, side-by-side, полный ASE 2016, отчёты)

---

## Фазы поставки MVP

### Полный MVP (целевое состояние)
DICOM + MP4 + JPEG/PNG · 2D viewer + side-by-side (`MasterClock`) · допплер PW/CW/TDI (AT, DT, IVRT, VTI, Vpeak, Vmean, PGpeak, PGmean, E/A, e', E/e') · стенки, LVOT · Тей-Хольц · Симпсон + Area-Length (ЛЖ, ЛП, ПП) · TAPSE, s' ПЖ, FAC · ONNX + сплайн · диастолика ASE 2016 · парсинг ЭКГ · PDF + CSV/JSON

### Фаза 1
DICOM + MP4 + JPEG/PNG · 2D viewer · допплер (все перечисленные параметры) · стенки, LVOT · Тей-Хольц · Симпсон + Area-Length с **ручным контуром** · ED/ES hotkeys

### Фаза 2
Side-by-side + `MasterClock` · ONNX + сплайн · ЛП/ПП/ПЖ · полный ASE 2016 (LVM, LAVmax, TR Vmax, градации) · парсинг ЭКГ · PDF/CSV/JSON · AI-валидация

Детали — в `Этап 1.md` §2.

## Ключевые принципы разработки

1. **Клиническая терминология** ASE/EACVI
2. **Локальная работа** — папки / сетевые диски, без DICOMweb
3. **Ручной контроль допплера** — пиковые и интервальные маркеры, VTI-трассировка (без авто-огибающей)
4. **Поэтапная поставка** — Фаза 1 без AI, Фаза 2 с EchoNet Segmentation Lite и отчётами
5. **Clean Architecture** — domain dataclasses, без pydicom в Domain
6. **Единый PyQtGraph** — без QGraphicsView
7. **CPU-only** ONNX *(Фаза 2)*
8. **Без RUO-ограничений** — нет водяных знаков и дисклеймеров
